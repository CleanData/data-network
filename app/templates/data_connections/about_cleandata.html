About Clean.Data.

Analysts and data scientists spend most of their time cleaning data. What we realised was that a lot of that time is spent cleaning the same data. What if you could find the data someone else has already cleaned?

Suddenly the problem isn't about data cleaning, but instead about data discovery. If you can find the data someone else has already cleaned, you save hours, weeks or months of duplicated effort.

What's in it for data scientists?

Every data scientist responsible for datasets has a profile page. If your datasets are highly used - or if you use a popular dataset in unusual ways - Clean.Data. will highlight that.

Physicists live by their citation count. Developers reference the number of times their Github project has been forked. Data Scientists should be able to point to the projects that rely on their data, and show off the value they add to open datasets. Now you can.

What's in it for industry?

Datasets don't need to be public to be represented in Clean.Data. If you have a commercial dataset that's used in applications around the world - add it to Clean.Data. When people ask about the value of the dataset, you can quickly reference who uses it, and where.

What's in it for government?

The open data movement is strong and growing stronger. But where is that data used? Clean.Data. offers an easy way to track and visualize the spread of open data. Tracking the use of open data is the first step to quantifying the value generated by open data.

How did Clean.Data. come about?

Clean.Data. originated at the New York Energy Week Data Jam in 2013. A team of nine people from industry, academia and government met and found that they shared a common problem finding clean datasets. The project was adopted as an initiative of Energy Infotech NYC.

The team from the Data Jam are Jon Roberts, Honey Berk, Sue Purvis, Barry Korn, Simon Sylvester-Chaudry, Matt Garcia, Tyler Wunsch, Mark Doolin and Mike Shimazu. The project is grateful for the support of Energy Solutions Forum.
